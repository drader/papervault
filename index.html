Agentic Knowledge Base

papervault

Drop arXiv PDFs or abstracts in. Agents extract contributions, methods, and limitations — linking authors, venues, and concepts into a growing research wiki automatically.

Built on memex  ·  github.com/drader/papervault
How it works

Drop a paper. Get a wiki.

1
Drop a paper
Place an arXiv PDF or plain-text abstract into wiki/raw/papers/. The daemon picks it up on the next scheduled cycle.
2
Agent reads and extracts
The researcher agent identifies the problem, contributions, methodology, results, and limitations. It resolves author names, venue details, and referenced methods.
3
Wiki pages are written
A structured paper summary, author profiles, method pages, concept pages, and venue pages are created and cross-linked bidirectionally.
4
Orchestrator promotes patterns
Every Monday the orchestrator scans agent memory for recurring patterns, promotes them to synthesis pages, and flags orphans and missing citations.
Wiki structure

One page per concept, bidirectionally linked.

wiki/
  raw/papers/       drop arXiv PDFs or .txt abstracts here
  papers/           one structured summary per paper
  authors/          researcher profiles, linked to all their papers
  methods/          algorithms and techniques introduced or referenced
  concepts/         higher-level research area ideas
  venues/           conferences and journals
  syntheses/        literature reviews, cross-paper comparisons
  archive/          retired pages (never deleted)

Every page links back to its sources. Authors link to papers; papers link to methods and venues. When the queue is empty, the agent runs a Wakeup routine — scanning for missing links, flagging gaps, and writing a "papers to consider" note to the journal.

Quick start

Up in three commands.

# clone and install
git clone https://github.com/drader/papervault my-papervault
cd my-papervault && npm install

# intake interview — seeds the wiki with your research area
npm run setup

# start the daemon
npm start

Then drop any arXiv PDF or abstract into wiki/raw/papers/ — or run /ingest inside Claude Code to process immediately. The daemon runs Tue–Fri 09:00 by default.

Slash commands

Everything through Claude Code.

CommandWhat it does
/ingestProcess the next unprocessed paper in wiki/raw/papers/
/queryAsk a research question — get a wiki-backed answer with paper citations
/lintCheck wiki health: orphan pages, missing citations, broken links
/setupFirst-time intake interview — seeds research area, authors, open questions
/session-startLoad context, log session open
/session-endFlush scratch notes, log session close
